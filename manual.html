1. ภาพรวมระบบ (System Overview)

ระบบ PDI System เป็นระบบจัดการสต็อกรถยนต์แบบครบวงจร ประกอบด้วยโมดูลหลัก 6 ส่วน:

  • 🤖 Telegram Bot (main.py): นำเข้า Excel, Stock Out/In, ค้นหา VIN, รายงาน
  • 🔧 PDI Web (Port 9000): คิว PDI, แจ้งซ่อม, แบตเตอรี่, VDCI Report
  • 📦 Inventory Web (Port 9111): จัดการสต็อก, ย้าย Slot, ตรวจนับ
  • 📊 Report Service (Port 9112): ออกรายงาน CSV/Preview พร้อมฟิลเตอร์
  • 📋 Claim Management (Port 9120): จัดการ Claim, สร้าง PDFรายงาน
  • 🔋 EDC Battery PDF (Port 7100): สร้าง PDF รายงานแบตเตอรี่ UT675A

2. การติดตั้งและตั้งค่า

โฟลเดอร์รันงาน (สำคัญมาก)

ทุกระบบจะอ่านและเขียนข้อมูลลงฐานข้อมูลเดียวกัน stock.db ดังนั้น ต้องเปิด terminal และรันคำสั่งต่างๆ จากโฟลเดอร์ที่มีไฟล์ระบบรวมกันอยู่

คำสั่งรันแบบ Manual

# Telegram Bot
python main.py

# PDI Web
python pdi_web.py

# Inventory Web
uvicorn inventory_web:app --host 0.0.0.0 --port 9111

*สามารถใช้ไฟล์ `start_system.bat` รันทุกระบบพร้อมกันได้เลย

3. Telegram Bot — คำสั่งทั้งหมด

คำสั่งการทำงาน
/startดูคำสั่งทั้งหมด
ส่งไฟล์ .xlsxอัปเดตข้อมูลรถเข้าสต็อก (ข้อมูลจาก Excel)
ส่งไฟล์มีคำว่า stockoutอัปเดต Stock Out พร้อมประทับเวลา
/s <VIN>ค้นหา VIN จากข้อความเลขท้าย 4-5 หลัก
/out <VIN>ทำ Stock Out (ถ้ารถติดล็อค PDI/Damages ทำไม่ได้)
/rein <VIN>Stock In รอบที่ 2 (รถที่เคยออกไปแล้ว)
/invดึง Excel สรุปสต็อกคงเหลือ
/otodayรายงานรถที่ออกไปในวันนี้
/PDMO <VIN>ส่งเข้าคิว PDI (ล็อครถ)
/PDMI <VIN> <Slot>ปลดล็อค PDI รถและระบุ Slot ใหม่ (PDI ต้อง 100%)
/EXPCF <VIN>ยืนยัน Export ชิปเมนต์

4. PDI Web — Port 9000

หน้าจัดการคิวและรายละเอียด PDI http://localhost:9000

  • PDI Queue: อัปเดตแบบเรียลไทม์ กดเปิด Job เพื่อกดยืนยัน FILL, VDCI, BODY
  • งานแจ้งซ่อม: สร้างงานแจ้งซ่อมใหม่ (จะล็อครถ) → เมื่อแก้เสร็จกด OK รถจะปลดล็อค
  • VDCI Report: อัปโหลดไฟล์ .html ก่อนและหลังแก้ เพื่อให้ระบบประเมินผลและปลดล็อคให้อัตโนมัติ
  • Batch Upload: ใส่โฟลเดอร์ VDCI ให้ระบบไล่ Process ทุกไฟล์ทีเดียว

5. Inventory Web — Port 9111

หน้าจัดการคลังรถยนต์และ Stock http://localhost:9111

  • ค้นหา: พิมพ์อักษรบางส่วนแล้วเลือกรถ แสดงรายละเอียดทั้งหมด (Slot, Lock, Status, Battery)
  • จัดการ Slot: เปลี่ยนตำแหน่งจอดได้ทันทีในหน้ารายละเอียด
  • ตรวจนับสต็อก: เปิด Job ขึ้นมา แล้วใช้เครื่องแสกน แสกน VIN ทีละคันลงระบบเพื่อนำเข้า Stock แบบแม่นยำ

6. Report Service — Port 9112

ระบบสร้างรายงานและส่งออกข้อมูลออกไปใช้งาน http://localhost:9112

  • ใช้ฟิลเตอร์ทั้ง 8 ช่อง เช่น ค้นหารถที่ยังไม่ PDI, รถที่มี/ไม่มี Slot หรือกรองตามสถานที่จะส่งออก
  • รองรับการดู Preview บนหน้าจอทันที หรือ Export ออกเป็น CSV

7. Claim Management — Port 9120

ระบบจัดการแบตชำรุดหรือขอเอกสาร Claim รถ http://localhost:9120

  • ระบุผล Battery CCA, SOC, SOH พร้อมอัปโหลดรูปแบตเตอร์รี่
  • สร้าง PDF ออกมาเป็นการรวมร่างระหว่าง Template บริษัท (เช่น รุ่น YP / ES), ค่าต่างๆ และภาพถ่าย ออกเป็น 1 ไฟล์สมบูรณ์

8. EDC Battery PDF — Port 7100

ระบบ Print PDF รายงานแบตเตอรี่หน้าตาเรียบง่าย http://localhost:7100

  • มีทั้งตัวเลือก Single และรูปแบบ Batch Upload สำหรับแสกนและออก PDF พร้อมกันหลายคันจาก Excel (ดึงค่า Dummy สุ่มที่มีความสมจริง)

9. Workflow และการใช้งานทั่วไป (Workflow)

  1. เริ่มต้นรับรถ: โยนไฟล์ Excel ของบริษัทเข้าแชท Bot จากนั้นลากรถไปจอดแสกน PDI
  2. ตรวจรับรถ: เข้า PDI Web สร้าง Job ทำตามขั้นตอน หรือ Upload VDCI → รถผ่านการ PDI จบ
  3. ตรวจนับ/Stock: เข้า Inventory Web ใช้เมนูแสกนนับรถเช็ค Slot ให้ถูกต้อง
  4. ปล่อยรถออก (Out): พิมพ์ /out VIN ใน Bot ถ้ายืนยันแล้วข้อมูลจะเข้า report

หากมีรถที่ต้องซ่อม ก็สร้างงานจากหน้า "งานแจ้งซ่อม" รถคันนั้นจะทำการ Out ไม่ได้จนกว่าจะกด OK.
